## The Honest Truth About Why This Exists

Most people learning to code have 50 browser tabs open, three half-finished courses, and zero shipped projects.

I was one of those people.

I kept switching — web dev one week, data science the next, mobile apps after that. I was busy but going nowhere. The kind of busy that feels productive until you look back and realize you haven't built anything real.

So I made a deal with myself on **Day 1**:

> *Pick one direction. Go all in. Build in public. Don't stop for 365 days.*

That direction is **AI / Machine Learning / AI Engineering.**

This repository is not a portfolio. It's not a course project dump. It's a **live record** of someone learning in public — the wins, the confusion, the "why isn't this working at 1am" moments, and the projects that come out the other side.

## My Building Philosophy

Three things I've decided to actually believe, not just say:

**1. Build before you're ready.**
Waiting until you fully understand something before building with it is how you never build anything. I write projects that are slightly beyond what I know. That gap is where learning happens.

**2. Consistency over intensity.**
A 4-hour coding binge followed by 5 days off is worse than 45 minutes every single day. This repo gets a commit as often as humanly possible.

**3. Clarity over comparison.**
Someone on Twitter just shipped an ML model at age 19. Someone else has been coding for a decade. Neither of those facts changes what I need to do today. This journey is measured against my Day 1 self — nothing else.

## Why Python First?

Python is the language of AI/ML. Period.

Before I touch a neural network, before I train a model, before I write a single line of NumPy — the fundamentals have to be solid. Syntax, data structures, OOP, file handling, error handling, functions — all of it.

Day 15 marks the end of Phase 1. The foundation is set.
